Project Overview

Built QuantBench, an end-to-end benchmarking pipeline for evaluating quantization strategies on transformer-based NLP models. The system evaluates four precision modes (FP32, FP16, INT8 Post-Training Quantization (PTQ) and INT8 Quantization-Aware Training (QAT)) on DistilBERT fine-tuned on the SST-2 sentiment classification dataset.

Each precision mode is measured across accuracy, inference latency, memory footprint and robustness under input corruption. A layer-wise sensitivity analysis identifies which of the 38 quantizable Linear modules can tolerate aggressive precision reduction. The key finding is that INT8-PTQ achieves 2.19x faster inference and 64% memory reduction with only 0.5 percentage point accuracy loss versus the FP32 baseline. Results are surfaced through an interactive Streamlit analytics dashboard (QuantBench).

Core Pipeline

1 Load distilbert-base-uncased-finetuned-sst-2-english (66M parameters, 6 transformer layers)
2 Apply four precision modes: FP32 baseline, FP16 half-precision, INT8-PTQ via dynamic quantization, INT8-QAT via fake-quantization fine-tuning
3 Benchmark each mode on 200-sample SST-2 validation subset measuring accuracy, avg latency, P99 latency and memory
4 Run robustness evaluation on 100-sample perturbed subsets: typo injection (10%), word dropout (15%), Gaussian noise (8%)
5 Perform layer-wise sensitivity analysis by quantizing each of 38 Linear modules individually and measuring accuracy delta vs FP32 baseline
6 Auto-detect hardware backend (fbgemm / qnnpack), validate quantized models, fall back to FP16 on failure
7 Surface all results in the QuantBench Streamlit dashboard with interactive charts, run history and a cost savings calculator

Benchmark Results

Results measured on 200 SST-2 validation samples. The FP32 baseline achieves 91.5% accuracy at 58.5 ms average latency using 255 MB of memory. FP16 matches FP32 accuracy but runs 4x slower on CPU due to lack of native FP16 compute. INT8-PTQ achieves 91.0% accuracy at 26.7 ms average latency using only 91 MB, making it the best overall tradeoff. INT8-QAT is the fastest at 22.3 ms but drops to 87.0% accuracy due to insufficient fine-tuning data.

Robustness Evaluation

FP32 and INT8-PTQ degrade nearly identically under all perturbation types. Typo injection (10%) drops both models by roughly 27 percentage points, character noise (8%) by around 19 points and word dropout (15%) by only 5 points. The gap between the two modes never exceeds 1 percentage point, confirming quantization does not amplify brittleness under noisy inputs.

Layer Sensitivity Analysis

Each of the 38 quantizable Linear modules was quantized individually to INT8 with all other layers kept at FP32. The middle feed-forward layers carry the most risk: quantizing layer.2.ffn alone drops accuracy by 3.0 percentage points and layer.2 by 2.5 points. Layers 0, 3 and their FFN sublayers show moderate sensitivity at 1.0 point. Layers 4 and 5 along with the classifier and pre-classifier heads show zero sensitivity. Layer 1 is the only component that improves slightly under quantization, likely due to a regularisation effect.

Technical Implementation

  • INT8-PTQ: Applied torch.quantization.quantize_dynamic targeting only nn.Linear layers; weights converted to INT8 at load time with activation scale factors computed at runtime with no calibration dataset required
  • INT8-QAT: Fake-quantization nodes inserted during fine-tuning on 300 SST-2 samples for one epoch using AdamW; model adapts weights to INT8 before final dynamic quantization conversion
  • Cross-Platform Handling: Pipeline auto-detects hardware backend (fbgemm on Intel x86, qnnpack on Apple Silicon), validates each quantized model with a test forward pass and falls back to FP16 on failure
  • Unit Testing: 24 pytest tests across 6 test classes (TestMetrics, TestPerturbations, TestMemory, TestQuantization, TestSensitivity and TestExport) with all 24 passing

Key Features

  • Multi-Mode Benchmarking: Unified pipeline comparing FP32, FP16, INT8-PTQ and INT8-QAT on accuracy, latency (avg + P99), memory and robustness in a single run
  • Layer Sensitivity Map: Per-layer INT8 quantization reveals which modules are safe to compress; layers 4, 5, classifier and pre_classifier show zero sensitivity
  • Robustness Testing: Three text perturbation types (typo injection, word dropout, character noise) confirm quantization does not increase model fragility
  • QuantBench Dashboard: Streamlit app with five sections (Benchmark Report, Run History, Robustness, Sensitivity and a Cost Calculator) projecting monthly and annual savings from switching to INT8-PTQ

Challenges & Solutions

  • Static Quantization Incompatibility: Static quantization crashes on nn.Embedding and nn.LayerNorm in PyTorch; solved by switching to dynamic quantization targeting only nn.Linear layers which is also hardware-portable
  • QAT Accuracy Drop: 300-sample fine-tuning set was insufficient for convergence, causing 4.5 pp accuracy loss; solution identified is training on the full 67,000-sample dataset with more epochs
  • Cross-Platform Backend Differences: Intel x86 uses fbgemm while Apple Silicon requires qnnpack with limited support; addressed via auto-detection with validated fallback to FP16

Results & Impact

INT8-PTQ is the recommended production deployment mode: 91.0% accuracy, 26.7 ms average latency (2.19x faster than FP32) and only 91 MB memory footprint (64% reduction from 255 MB). The gap between FP32 and INT8-PTQ under all perturbation types never exceeds 1 percentage point, confirming quantization introduces no additional brittleness.

The layer sensitivity analysis provides a concrete deployment strategy: keep layer.2.ffn and layer.2 at FP16 (highest risk layers) and quantize everything else to INT8. Layers 4, 5, the classifier and pre-classifier heads show zero sensitivity and can be safely compressed. The overall conclusion is that DistilBERT can be deployed at INT8 precision with confidence and at meaningfully lower infrastructure cost.
